    python думает, что запускается локальный
windows:
    1) python -m venv venv - создание
    2) venv\Scripts\activate - активация
    3) deactivate - деактивация

    *) pip freeze - показывает установленные пакеты
    *) pip install -U <name> - обновление какого-то модуля до последней версии
       pip install -U pip setuptools
    *) pip uninstall <name> -y - удаление (-y флаг)
    *) pip freeze > requirements.txt - создание файла зависимостей
    *) pip install -r requirements.txt

linux:
    1) python3 -m venv venv - создание
    2) source venv/bin/activate - активация (должно появится имя активации)
    3) deactivate - деактивация

    *) pip freeze - показывает установленные пакеты
    *) pip install -U <name> - обновление какого-то модуля до последней версии
       pip install -U pip setuptools
    *) pip uninstall <name> -y - удаление (-y флаг)
    *) pip freeze > requirements.txt - создание файла зависимостей
    *) pip install -r requirements.txt
